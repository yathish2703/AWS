What are security & identity services?

AWS security services provide comprehensive protection for your cloud infrastructure. From identity management to encryption, AWS offers tools to secure your applications, data, and users according to industry best practices and compliance requirements.

Main security services

IAM (Identity and Access Management)

Control access to AWS services and resources securely

Key features:

  • Fine-grained access control with policies
  • Multi-factor authentication (MFA)
  • Identity federation (SAML, SSO)

Components:

  • Users - Individual identities
  • Groups - Collections of users
  • Roles - Temporary credentials
  • Policies - Permission documents (JSON)
Learn more Global Service

Security Groups

Virtual firewalls that control inbound and outbound traffic to EC2 instances

Key features:

  • Stateful firewall (tracks connections)
  • Allow rules only (implicit deny)
  • Separate inbound and outbound rules

Common ports:

  • 22 - SSH (Linux remote access)
  • 80/443 - HTTP/HTTPS (Web traffic)
  • 3389 - RDP (Windows remote access)
  • 3306/5432 - MySQL/PostgreSQL
Learn more VPC-level

KMS (Key Management Service)

Create and control encryption keys for your data

Key features:

  • Centralized key management
  • Integrated with most AWS services
  • Audit key usage with CloudTrail

Common use cases:

  • Encrypt EBS volumes and S3 buckets
  • Protect database credentials
  • Sign and verify digital signatures
Regional Service

Secrets Manager

Securely store and manage database credentials, API keys, and other secrets

Key features:

  • Automatic rotation of credentials
  • Encrypted at rest using KMS
  • Fine-grained IAM access control

Common use cases:

  • Database passwords and connection strings
  • API keys and OAuth tokens
  • SSH keys and certificates
Regional Service

WAF (Web Application Firewall)

Protect web applications from common web exploits and bots

Key features:

  • Customizable web security rules
  • Managed rule groups (OWASP Top 10)
  • Real-time threat monitoring

Protection against:

  • SQL injection attacks
  • Cross-site scripting (XSS)
  • DDoS attacks and bad bots

AWS security best practices

Identity & Access

  • • Enable MFA for root account and privileged users
  • • Apply principle of least privilege
  • • Use IAM roles instead of access keys
  • • Rotate credentials regularly
  • • Never share credentials or access keys

Data Protection

  • • Encrypt data at rest and in transit
  • • Use KMS for key management
  • • Enable versioning on S3 buckets
  • • Implement backup and recovery plans
  • • Use VPC endpoints for private connectivity

Network Security

  • • Configure security groups with minimal access
  • • Use Network ACLs for subnet-level protection
  • • Enable VPC Flow Logs for monitoring
  • • Deploy WAF on public-facing applications
  • • Use private subnets for databases

Monitoring & Compliance

  • • Enable CloudTrail for audit logging
  • • Use AWS Config for compliance checks
  • • Set up GuardDuty for threat detection
  • • Monitor with Security Hub
  • • Regular security assessments and reviews

Security is a shared responsibility

AWS is responsible for security OF the cloud (infrastructure, hardware, software, networking). You are responsible for security IN the cloud (data, applications, IAM, encryption, network configuration).
