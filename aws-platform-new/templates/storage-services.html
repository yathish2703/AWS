Storage services

S3 (Simple Storage Service)

Object storage service with 99.999999999% (11 9's) durability

Key features:

  • Unlimited storage capacity
  • Storage classes for different access patterns
  • Versioning and lifecycle policies

Common use cases:

  • Data lakes and big data analytics
  • Backup and disaster recovery
  • Static website hosting

Storage Classes:

S3 Standard
S3 Intelligent-Tiering
S3 Glacier
S3 Glacier Deep Archive
Regional Service

EBS (Elastic Block Store)

High-performance block storage for EC2 instances

Key features:

  • Persistent block storage
  • Point-in-time snapshots
  • Encryption at rest

Volume types:

  • gp3/gp2 - General Purpose SSD
  • io2/io1 - Provisioned IOPS SSD
  • st1/sc1 - Throughput Optimized HDD
AZ-Specific

EFS (Elastic File System)

Scalable, elastic file storage for use with EC2 and on-premises

Key features:

  • Automatically scales as you add/remove files
  • Can be accessed by multiple EC2 instances
  • NFS v4 protocol support

Common use cases:

  • Content management systems
  • Shared development environments
  • Container storage
Regional Service

Database services

RDS (Relational Database Service)

Managed relational database service for MySQL, PostgreSQL, Oracle, SQL Server, MariaDB, and Aurora

Key features:

  • Automated backups and patching
  • Multi-AZ deployment for high availability
  • Read replicas for scalability

Supported engines:

  • Amazon Aurora (MySQL & PostgreSQL compatible)
  • MySQL, PostgreSQL, MariaDB
  • Oracle, Microsoft SQL Server
Regional Service

DynamoDB

Fast, flexible NoSQL database with single-digit millisecond latency

Key features:

  • Fully managed, serverless
  • Automatic scaling up and down
  • Global tables for multi-region replication

Common use cases:

  • Mobile and web applications
  • Gaming leaderboards
  • IoT and real-time applications
Regional Service

ElastiCache

In-memory caching service supporting Redis and Memcached

Key features:

  • Microsecond latency
  • Fully managed (patching, monitoring)
  • Automatic failover and recovery

Common use cases:

  • Session store for web applications
  • Database query result caching
  • Real-time leaderboards
Regional Service

Best practices

  • Choose the right storage type: Match storage service to your access patterns and durability needs
  • Enable encryption: Use encryption at rest and in transit for sensitive data
  • Implement backups: Regular automated backups for disaster recovery
  • Use lifecycle policies: Automate data movement to cost-effective storage tiers
  • Monitor performance: Use CloudWatch metrics to track storage and database performance
