Security Groups

Virtual Firewall for Your EC2 Instances

Control inbound and outbound traffic to protect your AWS resources

What are security groups?

Security Groups act as a virtual firewall that controls incoming and outgoing traffic for your EC2 instances. They work at the instance level, not the subnet level.

Deny by Default

All traffic is blocked unless explicitly allowed

Stateful

Return traffic is automatically allowed

Instant Changes

Rule changes take effect immediately

Inbound vs outbound rules

Inbound Rules

Control traffic coming INTO your instance

Type Port Source
HTTP 80 0.0.0.0/0
HTTPS 443 0.0.0.0/0
SSH 22 My IP
  • • Example: Allow HTTP (port 80) from anywhere
  • • Use case: Website traffic, SSH access

Outbound Rules

Control traffic going OUT from your instance

Type Port Destination
All traffic All 0.0.0.0/0
  • • Default: All outbound traffic is allowed
  • • Use case: Instance downloading updates, API calls

Common security group configurations

Web Server Security Group

For hosting public websites

✓ Inbound: HTTP (80) from 0.0.0.0/0
✓ Inbound: HTTPS (443) from 0.0.0.0/0
✓ Inbound: SSH (22) from Your IP only

Database Security Group

For RDS or EC2 database instances

✓ Inbound: MySQL (3306) from Web Server SG only
✗ No public internet access

Security best practices

Principle of least privilege

Only open ports you actually need

Restrict SSH access

Allow SSH only from your IP, not 0.0.0.0/0

Use descriptive names

Name security groups clearly (e.g., "web-server-sg")

Reference other security groups

Allow traffic between instances using SG IDs

Regular audits

Review and remove unused rules periodically

Never use 0.0.0.0/0 for databases

Databases should never be publicly accessible
