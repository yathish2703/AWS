What is EC2?

Amazon Elastic Compute Cloud (EC2) provides virtual servers called "instances" that you can use to run applications in the cloud. Instead of buying physical servers, you rent computing power from AWS and pay only for what you use.

Scalable

Scale up or down instantly

Cost-effective

Pay per hour/second

Secure

Built-in security features

EC2 instance types

Choose the right instance type based on your workload requirements

T

General Purpose (T3, T4g)

Balanced compute, memory, networking

  • Use case: Web servers, development environments
  • Example: t3.medium (2 vCPU, 4 GB RAM)
  • Cost: ~$0.04/hour
C

Compute Optimized (C6i)

High-performance processors

  • Use case: Gaming servers, scientific modeling
  • Example: c6i.large (2 vCPU, 4 GB RAM)
  • Cost: ~$0.09/hour
R

Memory Optimized (R6i)

High memory for data processing

  • Use case: Databases, in-memory caches
  • Example: r6i.large (2 vCPU, 16 GB RAM)
  • Cost: ~$0.13/hour
I

Storage Optimized (I4i)

High IOPS, low latency storage

  • Use case: NoSQL databases, data warehouses
  • Example: i4i.large (2 vCPU, 16 GB RAM, 468 GB NVMe)
  • Cost: ~$0.24/hour

How to host a website on EC2

Deployment flow

Your Code

EC2 Instance

Live Website

1

Launch an EC2 instance

Go to AWS Console → EC2 → Launch Instance

  • • Choose AMI: Amazon Linux 2 or Ubuntu Server
  • • Instance type: t3.micro (free tier eligible)
  • • Storage: 8 GB SSD (default)
2

Configure security group

Allow HTTP (port 80) and SSH (port 22) traffic

✓ SSH (22) - Your IP only
✓ HTTP (80) - 0.0.0.0/0 (anywhere)
✓ HTTPS (443) - 0.0.0.0/0 (anywhere)
3

Connect to your instance

Use SSH to connect to your EC2 instance

$ ssh -i my-key.pem ec2-user@54.123.45.67
4

Install web server

Install Apache or Nginx to serve your website

# Update packages
$ sudo yum update -y
# Install Apache
$ sudo yum install httpd -y
# Start Apache
$ sudo systemctl start httpd
$ sudo systemctl enable httpd
5

Deploy your website

Upload your HTML/CSS/JS files to the web server

# Navigate to web directory
$ cd /var/www/html
# Create your website
$ sudo nano index.html
# Your website is now live at http://your-ec2-ip

Key configuration options

Storage (EBS Volumes)

Persistent block storage for your instance

  • GP3: General purpose SSD (cost-effective)
  • IO2: High-performance SSD (databases)
  • Snapshots: Backup your volumes

Elastic IP

Static public IP address for your instance

  • Benefit: IP doesn't change on restart
  • Use case: Production websites
  • Cost: Free if attached to running instance

Auto Scaling

Automatically adjust instance count based on demand

  • Scale out: Add instances during high traffic
  • Scale in: Remove instances when idle
  • Cost savings: Pay only for what you need

User Data (Bootstrapping)

Script that runs when instance launches

  • Automate: Install software automatically
  • Example: Install web server on boot
  • Format: Shell script or cloud-init

Best practices

Always tag your instances for easy identification and cost tracking
Use IAM roles instead of storing credentials on the instance
Enable CloudWatch monitoring to track performance metrics
Regular backups using AMIs and EBS snapshots
